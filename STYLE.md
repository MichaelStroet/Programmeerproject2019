# Style Guide
### Michael Stroet - 11293284

### Comments
Comments worden geschreven voor blokken code van één of meerdere regels. Een blok bevat gerelateerde of samenhangende code die samen één doel hebben. Bij elke functie definitie wordt beschreven wat deze functie doet.

### Indentatie
Als indentatie worden er alleen spaties gebruikt. Elke laag is vier spaties wijd.

### Witregels
De bovengenoemde blokken zijn van elkaar gescheiden met één witregel. Elke functie onderling zijn ook gescheiden met een enkele witregel.

### Variabele/Functie namen
Elke variabele en functie heeft een naam die duidelijk omschrijft wat het is of wat het doet. Op de iteratie variabelen i, j, etc. na worden er geen enkele één-letter variabelen gebruikt.

### Code structuur
Elk bestand met code staat in een eigen map van de gebruikte programmeer taal (HTML/CSS, Javascript of Python). In deze mappen staan meerdere bestanden die elk een eigen doel hebben.
Bijvoorbeeld de hoofdbestanden stars.js en main.py, of de individuele bestanden voor elk figuur in de visualisatie.

### Patronen om te gebruiken
Hoe kleiner de scope van een functie hoe overzichtelijker de code wordt. Functies moeten over het algemeen één taak hebben die uitgevoerd wordt.

### Patronen om te vermijden
Het gebruik van functies waar veel variabelen aan meegegeven moeten worden, zorgt voor lange regels code die niet makkelijk leesbaar zijn.
